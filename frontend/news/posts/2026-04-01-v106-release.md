# v1.0.6 - Hidden Stats, Attachment Availablity, and Performance

**Released:** 2026-04-01

v1.0.6 introduces the hidden stats panel with **16** new per-weapon stats sourced from SPT and tarkov.dev API, and replaces the multi-request gun selection flow with a single batched API call for a noticeably faster gun load.

---

## Expanded Hidden Stats

The hidden stats panel now shows **16 stats**:

- **Aim Deviation** - base spread of the weapon when aiming down sights
- **Aim Sensitivity** - how sharply the camera responds to mouse input while aiming
- **Camera Angle Step** - increment at which the camera angle snaps per step
- **Camera Snap** - how quickly the camera returns to center after recoil
- **Deviation Curve** - the curve function applied to shot deviation over time
- **Max Deviation** - maximum possible deviation from point of aim
- **Mount Camera Snap** - multiplier for how aggressively the camera re-centers while mounted
- **Mount Horizontal Recoil** - recoil multiplier applied when firing from a mount, horizontal axis
- **Mount Vertical Recoil** - recoil multiplier applied when firing from a mount, vertical axis
- **Mount Breath Sway** - sway multiplier for out-of-breath penalty while mounted
- **Recoil Angle** - angle offset direction of the recoil pattern
- **Hand Rotation** - recoil hand rotation multiplier category
- **Recoil Dispersion** - spread applied to successive shots in the recoil pattern
- **Recoil Backwards Force** - physical force pushing the weapon back into the shooter on fire
- **Recoil Upwards Force** - physical force pushing the weapon upward on fire
- **Recoil Return Speed** - how quickly the weapon returns to its base position between shots

Stats are sourced from the latest SPT game files (from Tarkov base version Beta 0.16.9.0.40087) if the tarkov.dev API does not provide them. If a weapon has no entry in the data source, the value is shown as **?**. Values already provided by tarkov.dev continue to use that source and are not overwritten.

> **Note:** The descriptions above are my personal interpretation of each stat based on the field names and available documentation. Some may not be perfectly accurate - if you test something and find a description is wrong or misleading, please let me know!

## Gun Select Performance

Selecting a gun previously fired several sequential API calls: one per factory attachment, then a separate ammo request, then a calculate request. On weapons with many factory parts this could take a noticeable moment.

All of that is now replaced by a **single `/init` request** that returns the factory attachment tree, slot cache, ammo list, and initial build stats in one shot. Gun selection is substantially faster, particularly on slower connections.

---

## Local Saved Builds Limit Raised

The local saved builds limit has been raised from **50 to 500**. If you were hitting the cap and losing older saves, this should no longer be an issue.

---

## Trader Levels & Attachment Availability

The trader level system now extends to the attachment tree and selector:
- **Buyable filter** - A new **Buyable** toggle has been added to the attachment selector, next to the Compare toggle. When enabled, attachments that your traders cannot sell at your current loyalty levels are hidden from the list.
- **Quest unlock notes** - Attachments that require completing a trader task to unlock now show the task name below the attachment name in the selector.

---

## Bug Fixes

- Price chips in the attachment tree previously showed a trader price even when your loyalty level was too low to buy from that trader. They now correctly hide the trader price and fall back to the flea market price if one is available.
- Typing while the news drawer or leaderboard overlay is open no longer accidentally triggers the gun search bar in the background.
- Loading a community or shared build now shows a **loading overlay** on the slot panel while the build is being installed. Previously the old gun's slots would stay on screen briefly before being replaced. The overlay prevents that and gives a clear loading state.
- Fixed community cards not scaling properly in the Explore Builds modal while using Firefox.
