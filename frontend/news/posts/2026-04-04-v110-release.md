# v1.1.0 - Grid View Modding Interface

**Released:** 2026-04-04

v1.1.0 introduces the new grid view workbench - a visual attachment layout that mirrors the physical structure of your weapon - along with slot placeholder images, factory build fixes, and several QoL improvements.

---

## Grid View Workbench

The Attachment Tree panel (now renamed to **Workbench**) now has two view modes: **Grid** and **List**. Grid view is the new default.

In grid view, attachment slots are arranged spatially on a 2D canvas that roughly maps to the physical layout of the weapon (I tried my best😃👍) - barrel at the front, stock at the rear, optics on top, etc. Slots are grouped into zones (Upper, Lower, Left, Right, Extras), making it much easier to visualize what you're building at a glance.

![Grid View Demo](./news/images/v110/gridView.png)

You can toggle between the two view modes using the buttons at the top right of the Workbench panel!

*Special thanks to [db4tarkov](https://db4tarkov.com) for providing UI design inspirations <3<3*

---

## Slot Placeholder Images

Empty slots in both grid and list view now display a placeholder image representing the slot type instead of a plain `+` symbol. Each slot type (Barrel, Muzzle, Stock, Scope, Magazine, etc.) has its own placeholder icon, making it immediately clear what kind of attachment goes there.

---

## Deprecation Notice: EED and Overswing

The **EvoErgoDelta** and **Overswing** stats are now marked as deprecated following Tarkov patch 1.0.4.0.44026, which removed the underlying mechanics. They remain visible but are grouped and labeled accordingly.

---

## Bug Fixes & Misc

- Factory-default builds now (should?) load correctly in all cases. Previously, a race condition in the factory install order could result in a child item overwriting a parent that had already claimed a slot - leaving some slots in an invalid state. The backend now sorts items so parents are always processed first, and each slot is guarded so only the first match wins. The client-side reset also now replays from a stored snapshot rather than re-running the install logic.
- Rapidly opening different slots no longer causes out-of-order results to appear. Each `openSlotSelector` call now carries an incrementing token, and any response that arrives after a newer call has been made is silently discarded.
- Fixed conflict flashing failing to target the correct element in grid view (was using `getElementById` on a class-only element)
- When a conflicting item is the gun itself, the gun's grid cell now flashes instead of trying to find a slot
- Renamed "ATTACHMENT TREE" panel header to **"WORKBENCH"**
- Attachment flash animations now clear after re-render instead of before, preventing a brief flash-free frame
- `short_name` is now included in the gun list API response
