# v1.0.0 - 正式版发布

**发布日期：** 2026-03-23

v1.0.0 正式发布！这个项目从一个配件模拟器起步，如今已成长为一个完整的社区平台，涵盖实时价格、社区配枪、评分、排行榜、会话恢复等众多功能。本篇动态涵盖本次发布的全部四个开发里程碑的所有内容。

---

## 配件对比模式

![对比模式演示](./news/images/v100/compareMode.webp "右键选择「在新标签页中打开图片」以全屏查看！")

现在可以将任意已安装的配件设为对比基准。此后，在配件列表中将鼠标悬停于其他配件时，人体工学与后坐力条上将直接显示实时属性差值。重量与Evo人机工效差值也会同步在属性面板中实时更新。

无需反复切换配件，即可直观感受各配件之间的差异。

配枪界面现在在**配件**视图旁新增了**价格**视图，切换后可查看当前配枪中每件配件的详细费用明细，包括：

- **最低价格来源选择** - 每件物品自动显示商人与跳蚤市场中的最低价格，并附有商人头像与跳蚤图标以便区分
- **PvP / PvE 切换** - 在两套跳蚤价格缓存之间切换，无需重新请求数据
- **商人等级限制** - 可为Prapor, Skier, Peacekeeper, Mechanic和Jeager分别设置商人等级（LL1-4）；若某配件要求的等级高于当前设置，系统将自动回退至跳蚤价格。"全部"行可一次性设置所有商人，设置跨会话持久保存
- **配件树价格标签** - 配件树中的每行配件直接显示实时价格标签
- **跳蚤市场重新获取按钮** - 可随时手动刷新跳蚤价格，左侧显示上次获取时间

## 会话恢复

若您在配枪途中关闭或刷新了页面，配枪进度现在将自动恢复。返回后，上一把武器将以黄色高亮显示，再也不会因为误触刷新而丢失改好一半的枪！

这个系统和大家一样聪明！出厂配置以及舆已保存配枪完全匹配状态的配置将被排除在外，不会触发恢复提示。

切换语言时，完整的配枪状态也会在重新初始化后保留。

---

## 配枪状态智能识别

枪械显示名称现在会反映您保存的配枪。当前安装的配件与某个已保存配枪完全匹配时（与安装顺序无关），枪械名称将自动更新为该配枪的名称；当配件与之不符时，名称恢复为枪械本身的名称。

## 社区配枪

![社区配枪演示](./news/images/v100/communityBuilds.jpg "右键选择「在新标签页中打开图片」以全屏查看！")

配枪现在可以发布并与网站所有用户共享 😃👍！

**发布配枪**时将弹出确认面板，显示枪械图片与配枪名称，提交前可再次确认。发布的配枪在被接受前将经过实时物品数据库的校验。

**探索配枪对话框**并排显示当前选定武器的已保存配枪与社区配枪。社区配枪卡片显示作者头像、配枪名称、点赞按钮、加载次数、发布日期及配枪属性。加载社区配枪后，当前配件将被替换，作者署名会在占位区显示，直至您修改配枪。

**探索配枪按钮**显示已保存与社区配枪数量的实时合计标识，随保存或删除操作实时更新。

**删除已发布的本地配枪**时，系统将自动将其从社区列表中撤下并清除存储的服务器 ID。删除已发布配枪时会显示确认弹窗，避免误删社区内容。

## 排行榜

新增**排行榜**面板，共有两种模式：

- **社区配枪** - 近两周热门前 10 及全时段前 50，可按枪械筛选；点击任意条目可直接将其加载到配枪器中。发布您的配枪，有机会登上榜单！
- **配件** - 近两周热门前 20 / 全时段前 100，可按插槽类别筛选；支持按最多点赞或最多点踩排序

两种模式均采用密集排名（并列条目共享名次，无空缺），并按模式/周期/排序缓存结果，避免重复请求。

## 自定义界面控件

弹药类型与语言切换的原生 `<select>` 元素已替换为自定义下拉组件，带有国旗图标与展开/收起动画。对比模式与"假设满弹匣"切换按钮也已更新为自定义切换组件，与整体界面风格保持一致。

站点所有提示框现在统一通过单一自定义 `tooltip` 元素渲染，不再使用浏览器原生提示。任何带有 `data-tooltip` 属性的元素悬停时均会显示样式化提示框。Markdown 图片标题等原生 `title` 属性在首次悬停时自动转换，确保浏览器默认提示不会在站内任何位置出现。

任何按钮现在均可通过添加 `data-badge` 属性显示实时徽标，支持数字与文字内容，默认带有脉冲动画以吸引注意。探索配枪按钮即使用此功能显示已保存与社区配枪的合计数量，随操作实时更新。

## 代码清理与安全加固

这部分是给技术宅看的。v1.0.0 在发布前进行了一轮全面收紧：

- **可信代理 IP 配置** - 仅当直接连接来自已配置的已知反向代理 IP 列表（`TRUSTED_PROXY_IPS` 环境变量）时，才信任 `X-Forwarded-For` 与 `X-Real-IP` 请求头，防止通过请求头注入伪造 IP
- **输入长度限制** - 批量 ID 参数超过允许长度时将返回 `413`，阻止超大请求体
- **发布速率限制条目清理** - 速率限制映射表现在自动清理过期条目，防止持续流量下内存无限增长
- **`/health` 端点** - 存活检测与数据库连通性检查，供负载均衡器与在线监控使用
- **API 文档默认关闭** - 生产环境下 `/docs` 与 `/redoc` 默认隐藏，需设置 `ENABLE_API_DOCS=1` 才可访问，避免暴露完整 API 结构
- **前端定时器防重复注册** - 同步通知与时钟定时器现已加入防重复注册保护，避免重新初始化时重复注册
- **时钟 DOM 缓存** - 塔科夫时钟现在缓存每个条目的上次写入值，跳过无变化时的冗余 DOM 写入
- **语言切换保护** - 并发的语言切换调用现在被阻止，防止重新初始化期间的竞态条件
- **被动滚动监听器** - 面板拖拽缩放监听器现已注册为 passive，提升滚动性能
