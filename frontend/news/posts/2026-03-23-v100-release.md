# v1.0.0 - Full Release

**Released:** 2026-03-23

Release v1.0.0 is here! What started as a build simulator has grown into a full community platform with live pricing, community builds, ratings, leaderboards, session restore, and a lot more! This post covers everything across all four development milestones that make up this release.

---

## Attachment Compare Mode

<!-- IMAGE: A screenshot showing the compare baseline set on one attachment, with stat delta bars highlighted on another hovered attachment. Good single screenshot to anchor this section. -->

You can now set any installed attachment as a compare baseline. From there, hovering other attachments in the slot table shows live stat deltas directly on the ergo and recoil bars - green for improvements, red for losses. Weight and EED deltas update live in the stat panel at the same time.

This gives you a direct side-by-side feel without needing to swap attachments back and forth to see the difference.

## Price Panel

<!-- VIDEO: A short screen recording panning through the price panel - showing per-item rows with trader portraits, price chips in the attachment tree, the PvP/PvE toggle switch, and the flea refetch button. This is the single most visually dense new feature and a video communicates it better than a screenshot. Keep it under 30 seconds. -->

The build now has a full **Price** view alongside the **Build** view. Switching to it shows a per-item cost breakdown for every attachment in your current build, with:

- **Cheapest source selection** - each item shows the lowest price between trader and flea, with trader portraits and flea icon for clarity
- **PvP / PvE toggle** - switches between separate flea price caches without re-fetching
- **Trader loyalty level gating** - a per-trader level selector (LL1-4) for Prapor, Skier, Peacekeeper, Mechanic, and Jaeger gates trader prices to your configured loyalty level; if an attachment requires a higher tier than set, the panel falls back to flea automatically. An "All" row lets you set every trader at once. Settings persist across sessions
- **Price chips in the tree** - attachment rows in the build tree show live price chips at a glance without opening the price panel
- **Flea refetch button** - re-fetch flea prices on demand with a last-fetched timestamp shown below

## Session Restore

If you were mid-build and closed or refreshed the page, the build state is now recovered automatically. On return, a prompt shows the gun image and name and asks if you want to **Continue** or **Abandon** - no more losing a half-assembled build to an accidental F5.

The prompt is smart about it: factory configs and builds that already match a saved build are excluded since there's nothing meaningful to restore.

Language switching also preserves the full build state across reinit, with a toast confirming the restore.

---

## Build State Intelligence

The gun display name now reflects your saved builds. When the currently installed attachments exactly match a saved build (order-independent), the gun name updates to that build's name. When you deviate from it, the name reverts to the gun name.

---

## Community Builds

<!-- IMAGE: A screenshot of the Gun Builds dialog showing the saved builds panel and community builds panel side by side, with an author card visible. This is a clean layout screenshot that shows the two-panel design well. -->

Builds can now be published and shared across all users of the site.

**Publishing a build** opens a confirm panel with the gun image, a build name field, and an amber divider before submitting. Published builds are validated against the live item database before being accepted.

**The Gun Builds dialog** shows your saved builds and live community builds side by side for the currently selected weapon. Community build cards show the author avatar, build name, like/dislike buttons, load count, publish date, and build stats. Loading a community build replaces your current attachments and shows the author attribution in the placeholder until you modify it.

**The Builds button** in the header shows a live badge with your combined saved + community build count, updating as you save or delete.

**Admin tools** are available for featuring/unfeaturing builds, setting custom card images, deleting builds, managing author profiles, and banning clients. Banned clients are blocked from publishing until the ban expires or is lifted manually.

---

## Mid-Build Persistence Indicator

Unsaved mid-builds now survive clicking Back. The gun card gets a yellow border to signal that an in-progress build is waiting. Clicking that gun restores it silently. Clicking a different gun while a mid-build exists prompts an **Unsaved Changes** confirmation with the previous gun's image before switching.

---

## Leaderboard

The new **Leaderboard** panel has two modes:

- **Community Builds** - top 10 trending (last 2 weeks) and top 50 all-time, filterable by gun; clicking any entry loads it directly into the builder
- **Attachments** - top 20 trending / top 100 all-time, filterable by slot category; sortable by most liked or most disliked

Both modes use dense ranking (tied entries share a rank with no gaps) and cache results per mode/period/sort to avoid redundant fetches.

---

## Build Delete Flow

Deleting a local build that was previously published now automatically unlists it from the community feed and cleans up the stored server ID. A confirmation modal is shown when deleting a published build so you don't accidentally remove something from the community.

---

## Custom UI Controls

Native `<select>` elements for the ammo type and language switcher have been replaced with custom dropdowns featuring flag icons and animated open/close transitions. The compare mode and "assume full mag" toggles are now custom toggle components consistent with the rest of the UI.

---

## Tarkov Clock

The header now shows the current in-game Tarkov time for both servers. The left and right server clocks are always 12 hours apart. The clock updates every tick at 7x real time.

---

## Code Cleanup & Security Hardening

The v1.0.0 cleanup pass tightened several things before release:

- **Trusted proxy IP configuration** - `X-Forwarded-For` and `X-Real-IP` headers are now only trusted when the direct connection comes from a configured list of known reverse proxy IPs (`TRUSTED_PROXY_IPS` env var). Requests from unknown sources use the direct connection IP instead, preventing IP spoofing via header injection
- **Input length guards** - bulk ID parameters are now rejected with `413` if they exceed the allowed length, blocking oversized request payloads
- **Publish rate limiter eviction** - the rate limit map now evicts stale entries automatically to prevent unbounded memory growth under sustained traffic
- **`/health` endpoint** - liveness + DB connectivity check for load balancers and uptime monitors
- **API docs disabled by default** - `/docs` and `/redoc` are now hidden in production unless `ENABLE_API_DOCS=1` is set, avoiding full API schema exposure
- **Frontend interval guards** - the sync notice and clock intervals are guarded against double-registration across reinits
- **Clock DOM cache** - the Tarkov clock now caches the last written values per entry and skips redundant DOM writes on unchanged ticks
- **Language switch guard** - concurrent language switch calls are now blocked to prevent race conditions during reinit
- **Passive scroll listener** - the panel resize drag listener is now registered as passive for better scroll performance
