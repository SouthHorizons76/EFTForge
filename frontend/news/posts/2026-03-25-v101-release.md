# v1.0.1 - Mobile Iteration 1

**Released:** 2026-03-25

v1.0.1 marks the first iteration of the mobile compatibility optimizations for EFTForge. The site was originally built desktop-first, and this update begins closing the gap for phone users. More fixes will follow as we learn what works and what doesn't.

---

## Single-Panel Layout with Slide-Up Drawer

On any touch device, the two-panel desktop layout is replaced with a single full-width panel. The attachment selection table, which used to live in the right panel, is now a full-screen slide-up drawer that opens when you tap a slot and closes by tapping the backdrop or the back button inside the drawer.

The panel resizer is hidden on mobile, there is nothing to resize in a single-column layout.

---

## Landscape Two-Column View

In landscape orientation, the build panel splits into a two-column grid: build stats on the left, the attachment tree on the right. Both columns scroll independently so you can reach everything without the tree being buried under the stats section.

Rotating back to portrait collapses back to the single-column layout automatically.

---

## Swipe Left to Remove

Long-press to remove an attachment has been replaced with swipe left. Long-press on mobile almost always triggers the browser text selection popup before your handler fires, making it unreliable.

Swiping left on any filled slot in the tree or on an installed row in the attachment table reveals a red **REMOVE** strip. Release past the threshold to remove; let go early to cancel.

The system is self-registering via a MutationObserver - any element that opts in by adding a `swipe-removable` class and setting a `_swipeRemoveFn` property gets the gesture automatically, including any future features.

A `← swipe to remove` hint is shown inline with the attachment tree title on mobile so new users know the gesture exists.

---

## Publish on Mobile

The publish confirmation panel lives in the right-panel drawer, which stays closed during publish mode since there are no attachment slots to tap. To solve this, a gold `▲ REVIEW & PUBLISH` tray bar is pinned to the bottom of the screen when publish mode is active. Tap it to open the drawer and reach the confirm, modify, and cancel buttons. Tap the backdrop to dismiss and go back to reviewing the build. The tray disappears automatically after any outcome.

---

## Touch Polish

Several smaller improvements across the board:

- **Modals** - constrained to screen width with `max-height: 85vh` and internal scroll so they never overflow on any orientation
- **Toasts** - full viewport width, pinned to the bottom instead of the corner
- **Touch targets** - tree slot rows, toggle buttons, modal close buttons, option lists, and back buttons all have increased tap areas
- **Gun cards** - hover glow effects disabled on touch; `:active` highlight used instead
- **Search inputs** - forced to `16px` on portrait to prevent iOS auto-zoom on focus
- **Header** - clock hidden on portrait; site name kept visible
- **Attachment tree indentation** - reduced so deep trees fit on narrow screens without horizontal overflow
- **Community builds grid** - collapses to a single column on mobile
- **Scrollbars** - hidden on touch panels
